# Sight — UXPilot Master Design Prompt

## 1. Product Overview

Sight is a modern surveys and data analysis application. Users can:

* Answer surveys to earn points
* Create and publish surveys
* Analyze survey data using **SightAI** (core hero feature)
* Discover insights from aggregated and cross-referenced data

The product must feel:

* Intelligent, AI-driven, and trustworthy
* Fast-paced and intuitive
* Rich yet minimalist (no visual clutter)
* Modern consumer-tech quality (Gemini / ChatGPT level polish)

---

## 2. Brand & Design Language

### 2.1 Visual Personality

* Abstract, fluid, intelligent
* Soft gradients, smooth transitions
* Rounded geometry (cards, buttons, modals)
* Depth through subtle shadows and gradient layering
* No literal charts/icons as decoration — data is implied, not illustrated

### 2.2 Color Palette (Inspired by Logo)

Use these colors as inspiration across the app. Do not introduce new brand colors.

* Cyan / Aqua: `#35E0E6`
* Teal Blue: `#2BB6E9`
* Sky Blue: `#5FA9F5`
* Indigo Blue: `#4A63D8`
* Violet: `#8A4DE8`
* Purple: `#A23DD8`
* Magenta: `#D13DB8`
* Pink: `#FF6FAE`

#### Usage Guidance

* Backgrounds: white or very light blue-tinted white
* Primary actions: gradients using 2–3 adjacent colors
* Accents: pink / magenta / cyan
* SightAI: dark background with vibrant gradients from the full palette

---

## 3. Typography

* Clean, modern sans-serif
* High legibility
* Strong hierarchy

### Hierarchy

* App Name / Page Title: Bold
* Section Titles: Semibold
* Body Text: Regular
* Metadata (time, points, counts): Medium, muted color

---

## 4. Global UI Patterns

### 4.1 Buttons

* Primary: Gradient fill, rounded
* Secondary: Outline or subtle tinted background
* Destructive (archive/logout): soft red tint

### 4.2 Cards

* Rounded corners
* Light shadow
* Clear separation of content
* Use cards heavily for summaries and insights

### 4.3 Points System

* Always visible where relevant
* Represented as:

  * Small badge with gradient fill
  * Icon + numeric value
* Points gained per survey clearly shown

### 4.4 Bottom Navigation Bar (Global)

Persistent across all main pages:

* Home
* Surveys
* Research
* SightAI (center, larger, logo icon)
* Profile

## 8.2.2 Survey Answering Page

### Core Layout Rule (IMPORTANT)

* **All questions are stacked on a single scrollable page**
* Users scroll vertically to view and answer questions
* **No per-question pages, slides, or step-based flows**

### Question Presentation

* Each question appears as a card stacked vertically
* Clear spacing between questions
* Required questions visibly marked

### Question Types

* Written (text)
* Written (digits-only toggle)
* Choice:

  * Single choice
  * Multiple choice
  * If >6 options: inner scroll + search

### UI Behavior

* Inline validation (no blocking steps)
* Progress indicator reflects answered vs total questions (not steps)
* Submit button fixed at bottom or appears after last question

## 9. SightAI (Dark Mode Hero)

### Visual Style

* Dark background
* Vibrant gradients from full palette
* Elevated, premium feel

### Content

* Surveys already analyzed
* Mass analysis button (analyze other users’ surveys)
* Personal insights:

  * Cross-referenced analysis from answered surveys

### Analysis States

* Analyzing in background
* Strong visual indicator for:

  * In progress
  * Completed
* Re-analyze option when new responses are available

### Stats

* Number of analysis reports generated

---

## 9.1 Data Analysis Report Page

### Purpose

Display the finalized AI-generated analysis for a survey.

### Layout

* Header with survey title
* Metadata row:

  * Number of responses
  * Last analyzed date
  * Analysis version (e.g. v1, v2)

### Report Sections (Stacked)

* Key Insights (highlight cards)
* Trends & Patterns
* Notable Statistics
* Outliers or surprising findings

### Visualization Style

* Text-first insights
* Minimal abstract visuals or bars (non-interactive)
* Gradient accents only (no heavy charts)

### Actions

* Re-analyze survey (if new responses exist)
* View original survey

### UX Notes

* Content scrolls vertically
* Emphasis on clarity and readability
* Insights feel authoritative and confident

* Centered modal on desktop

### Content Structure

* Title (Privacy Policy / Terms & Conditions)

### UX Rules

* Content is read-only
* No checkboxes or decisions inside modal
* Modal can be dismissed via close icon or swipe down

## 11. Final Notes for UXPilot

* Do NOT add new features
* Do NOT invent backend logic
* Prioritize clarity and speed
* Everything should feel immediately understandable
* Light mode for all pages except SightAI
* Use gradients thoughtfully — rich but controlled
